# StudentHub 页面原型与 Figma 设计提示词 (Figma Prompt)

> **使用说明**：本提示词用于提交给 Figma AI、Galileo AI、Relume 或 UI 设计师。**本提示词不对视觉设计风格、色彩或字体做任何限定**，仅专注于**页面清单、页面布局结构、页面包含的核心元素及页面之间的跳转关系**。

---

## 1. 系统角色与信息架构

系统为高校“第二课堂”与学生事务管理中台，包含 4 类典型用户视角：**学生端**、**社团/自治干部端**、**辅导员/院系团委端**、**校级管理员端**。

全系统划分为 1 个统一登录页 + 1 个标准后台框架布局 (DefaultLayout) + 9 个一级导航模块 (30+ 核心视图页面)。

---

## 2. 全局通用框架结构 (Global Layout Structure)

在进入后台页面后，所有页面共享统一的后台框架 (DefaultLayout)，包含以下三大区域：

### 2.1 顶部导航栏 (Header)
* **左侧**：应用 Logo 图标、系统名称《StudentHub 学生一站式自主管理系统》。
* **右侧**：
  1. **身份切换器 (Role Switcher Dropdown)**：支持用户在“学生 / 团支部书记 / 社长 / 楼层长 / 辅导员 / 校团委”等角色间切换。
  2. **消息通知铃铛 (Notification Bell)**：带有未读消息数角标，点击展开未读通知下拉卡片，提供“一键已读”与“查看全部”。
  3. **用户个人中心下拉菜单 (User Profile Dropdown)**：显示用户头像与姓名，下拉项包含“我的档案”、“修改密码”、“退出登录”。

### 2.2 左侧侧边栏 (Sidebar Navigation)
动态两级/三级树状菜单，包含以下 7 大主域入口及对应图标：
1. 📊 **工作台** (Dashboard)
2. 🚩 **团员发展** (TY)
3. 🎪 **社团活动** (ST)
4. 🏢 **学生社区** (SQ)
5. 🛠️ **勤工助学** (QG)
6. 👤 **我的申请/个人中心** (Mine)
7. 🎓 **学生管理** (IDX)
8. ⚙️ **系统管理** (SYS)

### 2.3 主内容区 (Main Content Area)
* **顶端**：面包屑导航 (Breadcrumb) + 当前页面标题 + 右侧全局操作按钮区。
* **主体**：条件筛选表单区 + 数据展示区（表格/卡片/网格）+ 底部分页控制条 (Pagination)。

## 3. 核心页面清单与页面结构 (Page Structures)

### 3.1 登录页 (`/login`)
* **结构**：居中/分屏登录卡片。
* **元素**：系统标题、学号/工号输入框、密码输入框、记住密码复选框、“登录”按钮、底部版权与版本信息。

---

### 3.2 工作台与管理驾驶舱 (`/dashboard`, `/cmp/dashboard`)
* **结构**：多列响应式网格布局。
* **元素**：
  * **顶部统计卡片组**：在读学生数、待审批事项数、社团活动数、勤工在岗数。
  * **快捷操作区**：快速发起申请、新增活动、巡查打卡图标入口。
  * **数据大屏区**：ECharts 多维度统计图表（综合素质得分分布柱状图、社团活动类型饼图）。
  * **AI 综测智能助手卡片**：包含“一键生成学期综测评语”按钮、AI 评语文本展示框、改进建议列表及“人工覆写/保存”文本域。

---

### 3.3 综合分排行榜页面 (`/cmp/ranking`)
* **结构**：筛选栏 + 榜单表格。
* **元素**：学年学期选择器、院系/专业筛选下拉框、搜索框；表格包含名次（前三名带徽章）、学生姓名、学号、院系、团内分、社团分、社区分、勤工分、总分及导出 Excel 按钮。

---

### 3.4 团员发展模块页面群 (`/ty/*`)

#### (1) 入团申请列表与表单页 (`/ty/application`, `/ty/application/new`)
* **列表页**：筛选条件（申请状态、院系）、“提交申请”按钮；表格列出申请单号、姓名、学号、申请日期、当前节点、审批状态及“查看/编辑/撤回”操作。
* **表单页**：个人基础信息卡片、政治思想自述多行文本域（需显示字数统计，至少500字）、受奖励/处分情况、提交/保存草稿按钮。

#### (2) 审批中心页 (`/ty/approval`)
* **结构**：待我审批 / 我已审批 Tab 标签页。
* **元素**：申请单卡片列表，展开可查看学生申请全景与思想汇报，底部提供“同意/驳回”按钮及审批意见输入弹窗。

#### (3) 支部推优大会页 (`/ty/recommendation-meeting`)
* **结构**：新建会议弹窗/页面 + 会议记录列表。
* **元素**：团支部选择、会议时间/地点、应到人数、实到人数输入框；候选人投票表格（姓名、赞成票、反对票、弃权票、通过标记）；会场照片上传区（支持拖拽上传）。

#### (4) 培养考察与思想汇报页 (`/ty/cultivation`, `/mine/thought-report`)
* **结构**：学生培养卡片 + 思想汇报历史列表。
* **元素**：指定 2 名培养联系人信息；按季度展示思想汇报提交状态、汇报正文预览、AI 查重率结果标签及联系人考察意见填写框。

#### (5) 团员花名册与成长轨迹页 (`/ty/member-roster`, `/ty/students/:id/development-track`)
* **花名册**：全校团员搜索、团员证号检索、导出表格。
* **成长轨迹页**：时间轴 (Timeline) 组件，展示学生从“入团申请 ➔ 推优大会 ➔ 培养考察 ➔ 发展对象 ➔ 政审 ➔ 发展大会 ➔ 转正” 7 个阶段的时间节点、经办人与归档文件。

---

### 3.5 社团活动模块页面群 (`/st/*`)

#### (1) 社团管理与详情页 (`/st/association`, `/st/association/:id`)
* **列表页**：社团卡片网格，展示社团 Logo、名称、所属院系、指导教师、星级 (1-5星)、状态标签。
* **详情页**：社团概况、历任社长、招新计划 Tab、历史活动 Tab 及“发起换届/申请注销”按钮。

#### (2) 招新计划与申请广场 (`/st/recruit-plan`, `/st/recruit-apply`)
* **计划列表**：招新标题、目标人数、已录取人数、招新状态；提供“提前结束招新”按钮及确认弹窗（输入结束原因）。
* **广场页**：面向学生的招新海报卡片流，提供“立即投递”按钮与简历填写弹窗。

#### (3) 活动立项与分级审批页 (`/st/activity`, `/st/activity/approval`)
* **立项表单**：活动名称、社团选择、活动级别（A/B/C/D 自动计算）、预算金额、时间地点、应急预案文件上传组件 (MinIO) 及安全承诺书上传组件。
* **审批页**：展示审批流水链（指导教师 ➔ 院系 ➔ 校社联 ➔ 校团委 ➔ 校领导），高亮当前审批节点。

#### (4) 现场签到与活动总结页 (`/st/activity/:id/checkin`, `.../summary`)
* **签到页**：动态二维码展示框（定时刷新）、GPS 签到定位地图/提示、已签到学生实时列表。
* **总结页**：活动成果文本描述、现场照片多张上传墙、发票报销明细表单与附件预览。

---

### 3.6 学生社区与自治模块页面群 (`/sq/*`)

#### (1) 楼栋与寝室网格图页 (`/sq/building`)
* **结构**：左侧楼栋列表 + 右侧宿舍网格树 (Tree/Grid)。
* **元素**：楼栋卡片、楼层 Selector；寝室卡片（显示寝室号、床位使用情况、住宿学生头像及重点关注标记）、点击卡片弹窗展示床位调整与寝室长信息。

#### (2) 巡查记录大厅页 (`/sq/inspection`)
* **结构**：筛选条件 + 巡查记录流。
* **元素**：巡查类型切换 (卫生/晚归/违规电器/隐患)；巡查填报表单（扣分项勾选、违规照片拍照上传、晚归人员录入框）。

#### (3) 异常事件处置中心 (`/sq/incident`)
* **结构**：L1-L4 级事件看板 (Kanban) / 列表。
* **元素**：按 L1(常规报修)、L2(一般违规)、L3(严重违规)、L4(火警/突发) 颜色标签划分；事件详情卡片包含发生时间、地点、现场照片、协同处置日志，底部提供指导教师“一键结案”按钮。

---

### 3.7 勤工助学模块页面群 (`/qg/*`)

#### (1) 困难认定库页 (`/qg/difficulty`)
* **结构**：认定学年选择 + 困难学生列表。
* **元素**：困难等级标签 (特别困难/困难/一般困难)；材料预览按钮、班级评议记录、公示起止时间及导出清册按钮。

#### (2) 岗位管理与投递广场页 (`/qg/position`)
* **结构**：用人部门岗位卡片。
* **元素**：岗位名称、用人部门、时薪标准、每周工时上限、已录用/需求人数；困难生“申请上岗”按钮（非困难生显示置灰拦截提示）。

#### (3) 工时打卡与考勤页 (`/qg/attendance`)
* **结构**：打卡日历卡片 + 当月工时统计进度条。
* **元素**：上班打卡 / 下班打卡巨型按钮；**月累计工时进度条（满 40h 显示硬阻断红字提醒）**；补卡申请弹窗（支持双签审核状态展示）。

---

### 3.8 学生档案与系统管理页面群 (`/idx/*`, `/sys/*`)

#### (1) 学生列表与批量导入页 (`/idx/student`, `/idx/import`)
* **列表**：多条件组合查询、脱敏身份证/手机号展示；支持“查看全景档案”弹窗。
* **导入页**：Excel 文件拖拽上传组件、导入进度条、数据映射校验结果预览表及错误数据下载链接。

#### (2) 系统管理页面 (`/sys/dict`, `/sys/user`, `/sys/org`, `/sys/job`)
* **字典管理**：分类左树 + 右侧字典项配置表格。
* **用户管理**：用户列表、角色分配穿梭框 (Transfer)、重置密码弹窗。
* **组织管理**：院系-专业-班级树形编辑面板。
* **任务监控**：Cron 定时任务列表、执行状态 Tag、手动触发按钮与运行日志弹窗。

## 4. 页面跳转与交互流向关系 (Interaction Flows)

```text
[登录页 /login] 
       │ 认证通过
       ▼
[后台主框架 DefaultLayout] ➔ 默认跳转 [工作台 /dashboard]
       │
       ├── 点击侧边栏菜单 ➔ 切换主内容区页面 (/ty, /st, /sq, /qg, /idx, /sys)
       │
       ├── 列表页操作栏点击 [查看详情] ➔ 打开 [详情页 / 详情抽屉 Drawer]
       ├── 列表页操作栏点击 [编辑/新建] ➔ 跳转/打开 [表单页面 / Form Dialog]
       ├── 列表中点击 [文件预览] ➔ 弹出 [MinIO 预签名预览 Modal] (支持图片/PDF)
       ├── 涉及审批动作 ➔ 弹出 [审批意见填写 Dialog] (支持同意/驳回)
       ├── 工作台点击 [生成 AI 综测] ➔ 触发 [Spring AI 异步加载 State ➔ 呈现评语并支持覆写]
       │
       └── 顶栏点击 [身份切换器] ➔ 动态更新 [左侧侧边栏菜单项与页面权限按钮]
```
